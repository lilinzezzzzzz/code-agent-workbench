# AI 大模型交流最佳实践

本文整理一组可以直接追加在消息末尾的短句。适用于你已经给出背景、想法、
方案或内容后，要求 AI 给出更明确、更有批判性、更可执行的回答。

## 1. 核心原则

- **目标**：先围绕当前目标判断，不脱离目标做泛泛优化。
- **结论**：先明确回答可采用、需修改，还是不建议采用。
- **批判**：不要默认我的判断正确，有问题直接指出。
- **重点**：只列最重要的问题，并按优先级排序。
- **行动**：给出具体修改动作，不只描述问题。
- **验证**：给出判断建议是否有效的验收标准、测试方式或观察指标。
- **边界**：信息不足时明确说明，不自行补充事实。
- **取舍**：建议替代方案时说明收益、成本和风险。
- **标准**：判断是否符合最佳实践时，说明参考的原则、适用条件和例外情况。

## 2. 模型选择策略

- **全局默认**：Sol + Medium
- **临时升档**：
  - 复杂实现 / 排障：Sol + High
  - 高风险问题：Sol + Extra High
  - 最难单体问题：Sol + Max
  - 大型可拆分任务：Sol + Ultra
  - 大量明确、机械、多步骤工作：Terra + High 可能更划算

更高的 effort 有时只是增加探索和重复验证，并不必然改善最终代码。对 Sol
来说，**Medium 是最合理的能力利用起点，High 是最常用的质量升级档**。

## 3. 常见场景

### 方案 / 设计评审

```text
请评审这个方案是否适合采用。先给出明确结论，再按优先级指出关键问题、依据和改进建议。
```

```text
请重点检查这个设计是否存在不必要的复杂度、边界情况、失败路径和后续维护
风险；如果有更简单的替代方案，请比较当前方案和替代方案的收益、成本、
风险与适用条件，并明确推荐一个。
```

### 最佳实践符合性检查

```text
请判断这个做法是否符合当前领域最佳实践。先给结论，再说明依据、适用边界
和例外；如不符合，请说明影响、是否必须修改，以及最小可行改法。
```

### 风险 / 漏洞检查

```text
请从失败路径出发检查这个方案，列出最重要的 5 个风险，按严重程度排序，
并说明触发条件、影响和修复建议。
```

```text
请从攻击者视角检查可能的安全漏洞、权限绕过、数据泄露和滥用方式，并区分
现实风险与纯理论风险。
```

### 定向问题排查 / 修正

```text
请只检查下面的问题，不要扩展无关优化。先判断问题是否存在；如果存在，
给出最小修改和验证方式；如果不存在，说明判断依据。

问题：
xxxxxxx
```

### 内容精简 / 优化 / 重写

```text
不要改变核心目标，删除重复和低价值内容，补齐必要的逻辑缺口，然后直接给出优化后
的完整版本。新增或不确定的内容请单独标注。
```
