# AI 模型选择策略

官方建议使用能够满足结果要求的最低 reasoning effort；只有任务需要更深入的
规划、分析或检查时再逐级提高。更高的 effort 通常会增加延迟和 token 消耗，
不保证对所有任务都有更好的结果。

## 官方推荐起点

- **默认（Power）**：Sol + Medium。适合多数任务，也是官方推荐的默认组合。
- **日常均衡**：Terra + Medium。适合不需要 Sol 全部分析深度的常规工作。
- **快速、明确、可重复**：Luna + Light/Medium。适合目标和验收标准清晰的
  提取、分类、转换及结构化总结等任务。

## Effort 选择

- **Light**（CLI 中对应 Low）：适合快速、范围明确的简单任务。
- **Medium**：在速度与分析深度之间取得平衡，作为大多数任务的起点。
- **High / Extra High**：适合包含多个步骤、多个信息来源或重要权衡的困难
  任务，例如复杂实现、排障和设计评审。
- **Max**：让所选模型用更多时间处理单个任务。仅用于最困难、质量明显优先于
  延迟和用量的任务；若界面未显示该选项，需要先在应用设置中启用。
- **Ultra**：不是 reasoning effort。它会使用 subagents 并行处理复杂任务的
  不同部分，只适合能够拆分成多个有意义且相对独立工作流的大型任务。

## 个人代码工作策略

- **常驻默认**：Terra + High。默认优先保证代码实现、边界检查和验证质量，
  接受一定的额外延迟与 token 消耗，减少按任务频繁调整 effort 的成本。
- **复杂实现、跨模块排障、风险审查**：切换到 Sol + Medium/High
- **最困难的单体问题**：Sol + Extra High/Max。
- **大型且可并行拆分的任务**：Ultra。

Terra + High 是个人质量优先的常驻选择，不是官方面向所有用户的通用默认。
它可能让简单任务产生额外探索和重复验证，但在能够接受延迟与用量成本时，
可以换取更充分的规划和检查。多数任务不需要 Max 或 Ultra；不同模型版本的
effort 档位也没有严格的一一对应关系，模型升级后应使用熟悉的代表性任务重新
评估。
